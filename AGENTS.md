# oh-my-paul Agents

이 문서는 LLM이 oh-my-paul의 에이전트를 선택할 때 참고하는 가이드이다.

## 에이전트 목록

| 에이전트 | 모델 | 용도 | 파일 수정 |
|---|---|---|---|
| **reviewer** | claude-sonnet-4-6 | 코드 리뷰, 버그/보안/스타일 분석 | ❌ 읽기 전용 |
| **planner** | claude-sonnet-4-6 | 구현 계획 수립, 위험 분석 | ❌ 읽기 전용 |

## 선택 기준

### reviewer를 사용할 때
- 코드 리뷰 요청
- PR 리뷰
- 보안 취약점 분석
- 코드 품질 점검
- "리뷰해줘", "분석해줘", "이슈 찾아줘"

### planner를 사용할 때
- 복잡한 구현 전 계획 필요
- 리팩토링 범위 파악
- "계획 세워줘", "어떻게 구현할까", "설계해줘"

### 에이전트를 사용하지 않을 때
- 단순 코드 작성 → 직접 실행
- 파일 읽기/검색 → 직접 도구 사용
- 간단한 질문 → 직접 답변

## 에이전트 동작 규칙

1. 에이전트는 **독립된 프로세스**로 실행된다
2. 에이전트는 할당된 도구만 사용할 수 있다 (reviewer/planner는 Read, Glob, Grep만)
3. 에이전트는 **파일을 수정하지 않는다** — 분석/계획만 수행
4. 에이전트의 결과는 텍스트로 부모에게 반환된다
5. 에이전트는 다른 에이전트를 spawn하지 않는다
